# Superpowers + GStack 混合流程 v2.0

> 本 Skill 强制 AI 遵循"先规划、后评审、再编码"的严格流程
> 结合 Superpowers 的逻辑严密性和 gstack 的生产级标准
> **v2.0 新增**: 复杂度分级、冲突仲裁、性能基准、持续改进

```
用户: "请按照 superpowers-gstack-hybrid 流程执行"
```

## 任务复杂度分级

根据变更范围、影响面和风险等级，将任务分为三级：

### L1 - 简单任务 (轻量级流程)
**判定标准** (满足任一):
- 修改文件 < 3 个
- 新增代码 < 100 行
- 纯 bug 修复，无架构变更
- 文档更新

**简化流程**:
```
Phase 1 (简化) → Phase 6 → Phase 7 (简化)
```

**跳过**: Phase 2-5 的正式评审，改为快速自检查

### L2 - 中等任务 (标准流程)
**判定标准** (满足任一):
- 修改文件 3-8 个
- 新增代码 100-500 行
- 新增接口或模块
- 影响现有功能

**标准流程**:
```
Phase 1 → Phase 2-3 → Phase 6 → Phase 7
```

**跳过**: Phase 4-5 (QA/CSO 评审合并到 Phase 7)

### L3 - 复杂任务 (完整流程)
**判定标准** (满足任一):
- 修改文件 > 8 个
- 新增代码 > 500 行
- 架构重构
- 涉及安全或性能关键路径

**完整流程**:
```
Phase 1 → Phase 2 → Phase 3 → Phase 4 → Phase 5 → Phase 6 → Phase 7
```

```go
// 标准测试用例结构
func TestXxx(t *testing.T) {
    tests := []struct {
        name     string
        input    Input
        expected Expected
        wantErr  bool
        errType  error // 期望的错误类型
    }{
        {"正常情况", ...},
        {"空输入", ...},
        {"超大输入", ...},
        {"并发访问", ...},
        {"错误恢复", ...},
    }
    
    for _, tt := range tests {
        t.Run(tt.name, func(t *testing.T) {
            // 并行测试
            t.Parallel()
            
            got, err := Xxx(tt.input)
            
            if (err != nil) != tt.wantErr {
                t.Errorf("Xxx() error = %v, wantErr %v", err, tt.wantErr)
                return
            }
            
            if tt.wantErr && tt.errType != nil {
                if !errors.Is(err, tt.errType) {
                    t.Errorf("Xxx() error type = %T, want %T", err, tt.errType)
                }
            }
            
            if !reflect.DeepEqual(got, tt.expected) {
                t.Errorf("Xxx() = %v, want %v", got, tt.expected)
            }
        })
    }
}
```

## 冲突仲裁机制

当多角色评审意见冲突时：

### 冲突类型

| 冲突类型 | 示例 | 仲裁规则 |
|---------|------|---------|
| 架构 vs 性能 | 架构师要求分层，QA 担心性能 | 先满足架构，再优化性能 |
| 安全 vs 便利 | CSO 要求加密，产品经理要求快速 | 安全优先 |
| 质量 vs 进度 | QA 要求更多测试，项目要求按时交付 | 核心路径必须测试，非核心可后续补充 |

### 仲裁流程
```
冲突发生
    │
    ▼
识别冲突类型
    │
    ▼
应用仲裁规则
    │
    ▼
记录决策理由
    │
    ▼
更新 PLAN.md
    │
    ▼
通知相关方
```
